### 1. Создание и обработка массивов

| Функция | Реализация |
|---|---|
| `create_vector()` | `np.arange(10)` — генерирует целочисленный массив [0..9] |
| `create_matrix()` | `np.random.rand(5, 5)` — случайные числа из U[0,1) |
| `reshape_vector(vec)` | `vec.reshape(2, 5)` — изменяет форму без копирования данных |
| `transpose_matrix(mat)` | `mat.T` — транспонирование через атрибут |

!!! note "Нюанс"
    `reshape` не копирует данные — возвращается *представление* (view) исходного массива. Изменение элементов в результате `reshape` изменит оригинальный массив.

### 2. Векторные операции

Все операции выполняются **без Python-циклов** за счёт механизма векторизации NumPy — операции применяются поэлементно на уровне скомпилированного C-кода.

| Функция | Реализация |
|---|---|
| `vector_add(a, b)` | `a + b` |
| `scalar_multiply(vec, scalar)` | `vec * scalar` |
| `elementwise_multiply(a, b)` | `a * b` |
| `dot_product(a, b)` | `np.dot(a, b)` |

!!! note "Нюанс"
    `*` между массивами — поэлементное умножение, а не матричное. Для матричного произведения используется `@` или `np.matmul`.

### 3. Матричные операции

| Функция | Реализация |
|---|---|
| `matrix_multiply(a, b)` | `a @ b` (эквивалент `np.matmul`) |
| `matrix_determinant(a)` | `np.linalg.det(a)` |
| `matrix_inverse(a)` | `np.linalg.inv(a)` |
| `solve_linear_system(a, b)` | `np.linalg.solve(a, b)` |

!!! note "Нюансы"
    - `np.linalg.det` возвращает `float64`, поэтому для сравнения в тестах используется `round(..., 5)`.
    - `np.linalg.solve` работает точнее и быстрее, чем вычисление `A⁻¹ · b`, так как использует LU-разложение без явного вычисления обратной матрицы. Для вырожденных матриц выбрасывает `LinAlgError`.
    - Результат `matrix_inverse` проверяется условием `A @ A⁻¹ ≈ I` через `np.allclose`, а не точным равенством — из-за погрешностей floating-point арифметики.

### 4. Статистический анализ

#### `load_dataset(path)`

Данные загружаются через `pd.read_csv`, затем конвертируются в массив NumPy методом `.to_numpy()`. Это удобно, поскольку pandas автоматически обрабатывает заголовки CSV.

#### `statistical_analysis(data)`

Возвращает словарь с семью показателями:

| Ключ | Описание |
|---|---|
| `mean` | среднее арифметическое |
| `median` | медиана (устойчива к выбросам) |
| `std` | стандартное отклонение |
| `min` / `max` | экстремальные значения |
| `percentile_25` | нижний квартиль |
| `percentile_75` | верхний квартиль |

!!! note "Нюанс"
    NumPy по умолчанию вычисляет «популяционное» стандартное отклонение (делитель `N`), а не «выборочное» (делитель `N-1`). Если нужна несмещённая оценка, следует передать `ddof=1`.

#### `normalize_data(data)`

Применяет формулу Min-Max нормализации:

$$x' = \frac{x - x_{\min}}{x_{\max} - x_{\min}}$$

!!! warning "Нюанс"
    При константном массиве (все значения равны) знаменатель равен нулю — возникнет деление на 0.

### 5. Визуализация

Все функции:

1. Создают директорию `plots/` при необходимости (`os.makedirs(..., exist_ok=True)`).
2. Строят график в объектно-ориентированном стиле (`fig, ax = plt.subplots()`).
3. Сохраняют результат в PNG (`fig.savefig(...)`).
4. Закрывают фигуру (`plt.close(fig)`), чтобы не накапливать память при многократных вызовах.

| `plot_histogram(data)` | Гистограмма оценок по математике | `plots/histogram.png` |
| `plot_heatmap(matrix)` | Тепловая карта корреляции | `plots/heatmap.png` |
| `plot_line(x, y)` | Линейный график студент → оценка | `plots/line.png` |

!!! note "Нюанс"
    Тесты для визуализации проверяют только отсутствие исключений (`test_plot_*`), а не корректность самих изображений.
